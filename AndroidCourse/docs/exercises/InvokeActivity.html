Create a new activity in your project, ExplorerActivity:

  • right-click package; New; Other
  • in "Select a wizard", select Android/Android Activity; Next
  • Blank Activity with Fragment; Next
  • set Activity Name to ExplorerActivity; Next; Finish
Notice ExplorerActivity now added to AndroidManifest.xml
Notice ExplorerActivity has an inner class, PlaceholderFragment. Rename this to ExplorerFragment. We will discuss Fragments in more detail later in the course.

Add a button to end of MainActivity layout, to invoke ExplorerActivity; set the onClickListener, as we did for goButton.

In method onClick(...) determine which button has been clicked, by getting its id:

int view.getId(); // returns resource identifier

When the 'go' button is clicked, execute the code already in place for checking the supplied answer to the quiz; when the 'explorer' button is clicked, start the new activity:

Intent intent = new Intent(this, ExplorerActivity.class);
startActivity(intent);

Run your app. Ensure you can start ExplorerActivity by clicking the button. Use the back button to return to MainActivity.

In ExplorerActivity, override life-cycle callback methods:
onStart(), onResume(), onPause(), onStop(), onDestroy()

  • right-click on blank part of ExplorerActivity.java
  • choose Source, Override/Implement Methods

Override the same life-cycle methods, plus onCreate(), for ExplorerFragment.

Add logging to all the life-cycle callback methods, including Activity.onCreate() and Fragment.onCreateView(), e.g.

public final static String TAG = "ExplorerActivity";
...
@Override
protected void onStop() {
   if (BuildConfig.DEBUG) Log.d(TAG, "onStop()");
   super.onStop();
}

Make sure you use a different TAG for ExplorerFragment, and that every log message matches the method name.

In method Activity.onCreate() issue Toast message, e.g.

Toast.makeText(this, "ExplorerActivity.onCreate()",
Toast.LENGTH_LONG).show();

Run your app again in Debug mode, e.g. right-click project name in Project Explorer, and choose Debug as Android Application.

Look in log: open DDMS/LogCat view in Eclipse; in Saved Filters pane, click on the session filter for your app, identified by its package name; if there isn't one already created, add a filter, e.g.
  • click on green cross in Saved Filters
  • Filter Name: your package, e.g. com.john.student
  • by Application Name: your package
See what log messages you get when you rotate the device.
